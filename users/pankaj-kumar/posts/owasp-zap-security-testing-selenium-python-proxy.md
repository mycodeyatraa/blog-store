# Integrating OWASP ZAP with Selenium Python for Automated Security Testing

If you want to achieve true **DevSecOps**, you cannot rely solely on manual penetration testing. By the time a security engineer finds an SQL Injection vulnerability, the code has already been deployed.

What if your automated UI tests could actively scan for vulnerabilities *while* they navigate the application?

By integrating **OWASP ZAP (Zed Attack Proxy)** with Selenium, every single button click and page navigation performed by Selenium is intercepted and analyzed by ZAP in real-time. In this article, we will teach you how to route Selenium's traffic through ZAP and generate automated security reports!

---

## 1. What is OWASP ZAP?

OWASP ZAP is the world's most popular free, open-source security tool. It acts as a "Man-in-the-Middle" proxy. 

When you configure Selenium to use ZAP as its proxy:
1. Selenium attempts to click a button or load a page.
2. The network request goes to ZAP instead of the internet.
3. ZAP inspects the request for vulnerabilities (Missing CSRF tokens, XSS, SQLi).
4. ZAP forwards the request to the application.
5. ZAP inspects the response for missing security headers.

---

## 2. Setting up the ZAP Daemon

To run this automation, you need OWASP ZAP running in the background. You can download the desktop app, but in a CI/CD pipeline, you will run it via Docker:

```bash
docker run -u zap -p 8080:8080 -i owasp/zap2docker-stable zap.sh -daemon -port 8080 -host 0.0.0.0
```

This starts ZAP locally on `localhost:8080`.

Next, install the official ZAP Python API client:

```bash
pip install python-owasp-zap-v2.4
```

## 3. Routing Selenium Traffic Through ZAP

We need to tell `webdriver.Chrome()` to route all its traffic through our ZAP proxy running on port 8080.

**tests/test_zap_security.py**

```python
import time
from selenium import webdriver
from zapv2 import ZAPv2
# Configuration
ZAP_PROXY_HOST = "127.0.0.1"
ZAP_PROXY_PORT = 8080
TARGET_URL = "http://public-firing-range.appspot.com/" # A Google test site
def test_passive_security_scan():
    # 1. Configure Selenium to use the ZAP Proxy
    options = webdriver.ChromeOptions()
    options.add_argument(f'--proxy-server=http://{ZAP_PROXY_HOST}:{ZAP_PROXY_PORT}')
    # Ignore certificate errors since ZAP uses a self-signed cert to intercept HTTPS
    options.add_argument('--ignore-certificate-errors')
    driver = webdriver.Chrome(options=options)
    # 2. Connect to the ZAP API
    zap = ZAPv2(proxies={'http': f'http://{ZAP_PROXY_HOST}:{ZAP_PROXY_PORT}', 
                         'https': f'http://{ZAP_PROXY_HOST}:{ZAP_PROXY_PORT}'})
    # 3. Selenium navigates the app normally
    print("\n[Selenium] Navigating the application...")
    driver.get(TARGET_URL)
    # Imagine Selenium clicking through multiple pages here...
    time.sleep(3) # Give ZAP a moment to process the traffic
    driver.quit()
    # 4. Analyze the ZAP Results!
    print("[ZAP] Analyzing captured traffic for vulnerabilities...")
    # Fetch alerts specific to our target URL
    alerts = zap.core.alerts(baseurl=TARGET_URL)
    print(f"\nFound {len(alerts)} potential security vulnerabilities:")
    high_risk_count = 0
    for alert in alerts:
        risk = alert.get('risk')
        name = alert.get('name')
        if risk == "High":
            print(f"🚨 CRITICAL: {name}")
            high_risk_count += 1
        elif risk == "Medium":
            print(f"⚠️ WARNING: {name}")
    # 5. Fail the test if High Risk vulnerabilities exist!
    assert high_risk_count == 0, f"Found {high_risk_count} High Risk Vulnerabilities! Build Failed!"
```

---

## 4. Passive vs Active Scanning

The script above performs **Passive Scanning**. ZAP simply watches the traffic generated by Selenium and flags issues like missing security headers or exposed cookies. Passive scanning is incredibly safe and can be run in Production environments.

If you want ZAP to actively attack the application (submitting malicious SQL queries into search boxes), you can trigger an **Active Scan** via the API:

```python
# WARNING: NEVER run this against Production!
print("[ZAP] Initiating Active Attack Scan...")
zap.ascan.scan(TARGET_URL)
while int(zap.ascan.status()) < 100:
    print(f"Scan progress: {zap.ascan.status()}%")
    time.sleep(5)
print("Active Scan Complete!")
```

## Conclusion

Combining Selenium and OWASP ZAP is the ultimate CI/CD DevSecOps strategy.
- You do not need to write explicit security tests. Just run your existing UI tests through the ZAP proxy!
- ZAP will passively detect missing headers, exposed PII, and insecure cookies.
- You can query the ZAP API at the end of the test suite. If the `alerts` array contains any High-Risk vulnerabilities, immediately fail the Jenkins/GitHub Actions build!
