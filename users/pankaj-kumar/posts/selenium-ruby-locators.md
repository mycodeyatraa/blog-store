Welcome to Blog 3 of the **Selenium Ruby Mastery Series**! 

Now that you understand how Ruby communicates with the browser, it's time to actually interact with the UI. To click a button, type into a text box, or scrape text from a paragraph, you first need to find the element in the HTML DOM. 

Selenium provides 8 built-in **Locator Strategies**. In Ruby, these are passed as symbols (e.g., `:id`, `:xpath`) to the `find_element` method.

### The 8 Locator Strategies

1. **`:id`**: The fastest and most reliable locator. IDs are supposed to be unique per HTML specification.
2. **`:name`**: Very common on form inputs (`<input name="email">`).
3. **`:class`** (or `:class_name`): Used to find elements by their CSS class.
4. **`:tag_name`**: Finds elements by HTML tag (`button`, `a`, `div`).
5. **`:link_text`**: Finds `<a>` (anchor) tags by exact visible text.
6. **`:partial_link_text`**: Finds `<a>` tags by partial visible text.
7. **`:css`**: Highly flexible string-matching using standard CSS selectors.
8. **`:xpath`**: The most powerful locator, allowing you to traverse up and down the DOM tree.

### Practical Example: Automating a Login Form

Let's write an RSpec script that automates the login form on the [Live Sandbox Application](https://practice.mycodeyatra.com/#/login). We will deliberately use different locators for each field to see them in action.

Create a new file `spec/blog3_locators_spec.rb`:

```ruby
require 'selenium-webdriver'
require 'rspec'
RSpec.describe 'Blog 3: Mastering Locators' do
  it 'locates elements using various strategies and performs a login' do
    options = Selenium::WebDriver::Chrome::Options.new
    options.add_argument('--headless')
    driver = Selenium::WebDriver.for :chrome, options: options
    # 1. Navigate to the Login Page
    driver.navigate.to 'https://practice.mycodeyatra.com/#/login'
    # Wait for React to render the DOM (we'll cover smart waits in Blog 5)
    sleep 2 
    # 2. Locate by ID
    username_field = driver.find_element(id: 'username')
    username_field.send_keys('admin')
    puts "Found username field via :id"
    # 3. Locate by Name
    password_field = driver.find_element(name: 'password')
    password_field.send_keys('admin123')
    puts "Found password field via :name"
    # 4. Locate by CSS Selector
    # Let's verify our username field using CSS
    css_username = driver.find_element(css: 'input#username')
    expect(css_username.attribute('value')).to eq('admin')
    puts "Verified username value via :css"
    # 5. Locate by XPath
    login_button = driver.find_element(xpath: "//button[@type='submit']")
    login_button.click
    puts "Clicked the login button via :xpath"
    driver.quit
  end
end
```

### Breaking Down the Code

*   **`driver.find_element(id: 'username')`**: Ruby allows us to pass a hash `key: value` to `find_element`. The key is the locator strategy, and the value is the string to search for.
*   **`.send_keys`**: Simulates a user typing on their physical keyboard.
*   **`.click`**: Simulates a left mouse click.
*   **`.attribute('value')`**: A very useful command to extract the current text typed inside an `<input>` field, which doesn't actually exist as inner HTML text.
