CLI outputs are great for developers, but when it comes to sharing accessibility test results with Product Managers, QA Leads, or external auditors, a visual HTML report is far superior.

In this tutorial, we will learn how to generate beautiful, shareable HTML accessibility reports from our Playwright Axe-core scans using the `axe-html-reporter` package.

### Prerequisites

First, install the HTML reporter package alongside Axe-core:

```bash
npm install -D axe-html-reporter
```

### Generating the Report

The `axe-html-reporter` library exposes a `createHtmlReport` function that consumes the JSON `accessibilityScanResults` generated by `AxeBuilder` and transforms it into a standalone HTML document.

**File:** `tests/accessibility-reports.spec.ts`

```typescript
import { test, expect } from '@playwright/test';
import AxeBuilder from '@axe-core/playwright';
import { createHtmlReport } from 'axe-html-reporter';
import fs from 'fs';
 
test.describe('Accessibility Reporting', () => {
 
    test('should generate an HTML accessibility report', async ({ page }) => {
        // Navigate to the target page
        await page.goto('https://practice.mycodeyatra.com/');
        await page.waitForLoadState('networkidle');
 
        // Analyze the entire page
        const accessibilityScanResults = await new AxeBuilder({ page }).analyze();
 
        // Ensure a directory for reports exists
        if (!fs.existsSync('a11y-reports')) {
            fs.mkdirSync('a11y-reports');
        }
 
        // Generate the HTML report based on the scan results
        createHtmlReport({
            results: accessibilityScanResults,
            options: {
                projectKey: 'MyCodeYatra Practice App',
                outputDir: 'a11y-reports',
                reportFileName: 'accessibility-report.html'
            }
        });
 
        console.log(`[Reporting] Axe-core HTML report generated at: a11y-reports/accessibility-report.html`);
        console.log(`[Reporting] Total violations found: ${accessibilityScanResults.violations.length}`);
 
        // Asserting empty violations could fail the build here, 
        // but the report will still be generated for investigation!
    });
 
});
```

### Summary

Integrating `axe-html-reporter` adds tremendous value to CI/CD pipelines. By archiving the resulting HTML file as an artifact in Jenkins, GitHub Actions, or GitLab CI, stakeholders can simply click a link to view a comprehensive breakdown of all accessibility violations, the exact DOM nodes affected, and instructions on how to fix them!
