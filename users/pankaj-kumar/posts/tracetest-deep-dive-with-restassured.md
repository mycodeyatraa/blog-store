## 1. What is Tracetest?

[Tracetest](https://tracetest.io/) is a testing tool based on OpenTelemetry that allows you to assert against the internal telemetry of your microservices. 

With RestAssured, you assert what the user sees (the HTTP Response). With Tracetest, you assert what the system *did* (the Database Queries, the RabbitMQ messages, the third-party API calls).

## 2. Integrating Tracetest with RestAssured

Tracetest provides a seamless Java integration for RestAssured. It acts as an interceptor that listens to your test requests, captures the trace context, queries your telemetry datastore (e.g. Jaeger), and evaluates your assertions against the spans!

First, add the Tracetest dependency to your `pom.xml`:

```xml
<dependency>
    <groupId>io.tracetest</groupId>
    <artifactId>tracetest-restassured</artifactId>
    <version>1.0.0</version>
    <scope>test</scope>
</dependency>
```

## 3. Writing a Telemetry-Driven Assertion

Let's assume our `/api/users` POST endpoint is supposed to:
1. Return a `201 Created` HTTP Response.
2. Execute a specific `INSERT INTO users` query in PostgreSQL.
3. Publish a `user_created` event to Kafka.

In traditional testing, verifying steps 2 and 3 would require writing messy database connections or listening to Kafka topics. With Tracetest, it takes just a few lines of fluent code!

```java
import io.tracetest.restassured.TracetestFilter;
import org.testng.annotations.Test;
import static io.restassured.RestAssured.given;
public class TracetestDeepDiveTest {
    // Initialize the Tracetest Interceptor
    private final TracetestFilter tracetest = new TracetestFilter();
    @Test
    public void testUserCreationEmitsKafkaEvent() {
        String requestBody = "{ \\"name\\": \\"Pankaj\\", \\"role\\": \\"admin\\" }";
        given()
            .baseUri("http://localhost:8080")
            .filter(tracetest) // Inject Tracetest
            .header("Content-Type", "application/json")
            .body(requestBody)
        .when()
            .post("/api/users")
        .then()
            .statusCode(201); // 1. Assert the HTTP Response (RestAssured)
        // 2. Assert the Database Query (Tracetest)
        tracetest.assertThat()
            .span(span -> span.name("INSERT INTO users"))
            .hasAttribute("db.system", "postgresql")
            .hasDurationLessThan(50); // Query must execute in under 50ms
        // 3. Assert the Kafka Event (Tracetest)
        tracetest.assertThat()
            .span(span -> span.name("publish user_created"))
            .hasAttribute("messaging.system", "kafka")
            .hasAttribute("messaging.destination", "users-topic");
    }
}
```

## 4. The Magic of Observability

When you run this test, `tracetest.assertThat()` pauses execution for a few milliseconds, reaches out to your Jaeger/Datadog instance, fetches the exact trace generated by the RestAssured request, and evaluates your rules against the distributed spans.

If a developer accidentally changes the Kafka topic name from `users-topic` to `v2-users-topic`, the HTTP Response will still return `201 Created` (passing a traditional test), but **Tracetest will catch the invisible regression immediately!**
