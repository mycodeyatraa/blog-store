# Mastering SSO and MFA Authentication Flows in Selenium Java

In the modern enterprise, traditional username and password forms are increasingly rare. Today, users log in through external Single Sign-On (SSO) providers like Google Workspace, Okta, or Microsoft Entra ID. Furthermore, zero-trust architectures mandate Multi-Factor Authentication (MFA) via SMS codes or Authenticator apps.

These modern security protocols are excellent for protecting companies against hackers, but they are an absolute nightmare for UI Automation Engineers. 

In this article, we will teach you how to successfully automate SSO redirect flows and bypass MFA challenges using Selenium WebDriver in Java.

---

## 1. Single Sign-On (SSO) Redirect Flows

Testing an application that uses Okta or Google SSO means your WebDriver must navigate across multiple domains (`mycodeyatra.com` -> `accounts.google.com` -> `mycodeyatra.com`). 

### The Problem: Cross-Domain Cookie Loss
Selenium's `driver.manage().addCookie()` only works for the *current domain*. If you try to inject an authentication cookie for your internal app while sitting on the Okta login page, Selenium will throw an `InvalidCookieDomainException`.

### The Solution: The Wait-and-Catch Pattern
To test SSO, you cannot inject cookies directly. You must let the UI redirect to the SSO provider, perform the SSO login, wait for the redirect *back* to your application, and then capture the resulting token.

**Testing Google SSO (Simulated):**

```java
import org.openqa.selenium.By;
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.support.ui.ExpectedConditions;
import org.openqa.selenium.support.ui.WebDriverWait;
import org.testng.Assert;
import org.testng.annotations.Test;
import java.time.Duration;
public class SSOTest {
    @Test
    public void testOktaSSOLogin() {
        WebDriver driver = new ChromeDriver();
        WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(15));
        // 1. Navigate to your app
        driver.get("https://mycodeyatra.com/login");
        // 2. Click "Login with Okta"
        driver.findElement(By.id("okta-sso-btn")).click();
        // 3. Wait for redirect to Okta domain
        wait.until(ExpectedConditions.urlContains("okta.com"));
        // 4. Fill out Okta credentials (Ensure these are test accounts!)
        driver.findElement(By.id("okta-signin-username")).sendKeys("testuser@mycodeyatra.com");
        driver.findElement(By.id("okta-signin-password")).sendKeys("TestPass123!");
        driver.findElement(By.id("okta-signin-submit")).click();
        // 5. CRITICAL: Wait for the redirect BACK to your domain
        wait.until(ExpectedConditions.urlContains("mycodeyatra.com/dashboard"));
        Assert.assertTrue(driver.getPageSource().contains("Welcome back"));
        driver.quit();
    }
}
```

## 2. Multi-Factor Authentication (MFA) Concepts

When you log in, your application might ask for a 6-digit code sent via SMS or generated by an Authenticator app (TOTP). Selenium cannot read your personal cell phone. How do we test this?

### Strategy 1: The Dev-Environment Bypass (Recommended)
The absolute best way to handle MFA in automation is to **not test it at all in the UI layer**.
Work with your developers to add a backdoor in lower environments (QA/Staging). 
For example, developers can configure the QA environment so that the MFA code `000000` is universally accepted for designated Test User accounts.

### Strategy 2: Programmatic TOTP Generation (Authenticator App)
If you *must* test MFA in production, you can use Java libraries to programmatically generate the 6-digit Time-Based One-Time Password (TOTP) that Google Authenticator uses.

When you set up MFA for your test account, save the **Secret Key** (a long alphanumeric string) instead of scanning the QR code. You can then use the `aerogear-otp-java` library to generate the code in real-time!

**Adding Dependency (Maven):**

```xml
<dependency>
    <groupId>org.jboss.aerogear</groupId>
    <artifactId>aerogear-otp-java</artifactId>
    <version>1.0.0</version>
</dependency>
```

**TotpMFATest.java**

```java
import org.jboss.aerogear.security.otp.Totp;
import org.openqa.selenium.By;
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;
import org.testng.annotations.Test;
public class TotpMFATest {
    @Test
    public void testLoginWithAuthenticatorApp() {
        WebDriver driver = new ChromeDriver();
        driver.get("https://mycodeyatra.com/login");
        // Submit Password
        driver.findElement(By.id("user")).sendKeys("admin");
        driver.findElement(By.id("pass")).sendKeys("SuperSecret!");
        driver.findElement(By.id("submit")).click();
        // --- WE ARE PROMPTED FOR MFA ---
        // 1. Retrieve your Test Account's Secret Key (Store securely in Env Vars!)
        String secretKey = System.getenv("MFA_SECRET_KEY"); 
        // 2. Generate the live 6-digit code!
        Totp totpGenerator = new Totp(secretKey);
        String liveMfaCode = totpGenerator.now();
        System.out.println("Generated Live MFA Code: " + liveMfaCode);
        // 3. Inject it into the UI
        driver.findElement(By.id("mfa-input")).sendKeys(liveMfaCode);
        driver.findElement(By.id("mfa-submit")).click();
        // Assert Login Success...
        driver.quit();
    }
}
```

### Strategy 3: SMS Verification via Twilio API
If the system uses SMS, you can use a service like **Twilio**. 
1. Purchase a virtual phone number from Twilio.
2. Link the Test Account to the Twilio number.
3. When the test hits the MFA screen, use Twilio's Java SDK (or RestAssured) to fetch the most recent SMS message received by that number, parse the 6-digit code, and enter it into Selenium!

## Conclusion

SSO and MFA are designed to block automated bots, which means they naturally block Selenium. To overcome them:
- **For SSO:** Master `WebDriverWait`. Wait for the domain redirect to the provider, and wait for the redirect *back* to your app.
- **For MFA:** Ask developers for a static bypass code in QA. If that fails, generate TOTP codes programmatically using `aerogear-otp-java`, or intercept SMS messages using virtual numbers via Twilio API.
