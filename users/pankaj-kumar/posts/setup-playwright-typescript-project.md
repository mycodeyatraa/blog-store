Welcome to Blog 3 of the **Playwright TypeScript Mastery Series**!

Now that we understand the architectural superiority of Playwright and TypeScript, it is time to build our framework. Unlike Selenium, which requires you to manually download IDEs, JDKs, compiler plugins, and driver executables, setting up Playwright is incredibly streamlined.

Microsoft has provided a dedicated initialization command that automatically builds your entire project architecture in less than a minute.

### Step 1: Install Node.js

Because Playwright runs on Node.js, you must install it first.

1.  Navigate to [nodejs.org](https://nodejs.org/).
2.  Download the **LTS (Long Term Support)** version for your operating system.
3.  Install it with the default settings.

To verify your installation, open your terminal (Command Prompt, PowerShell, or macOS Terminal) and type:

```
node -v
npm -v
```

*Expected Output:* You should see version numbers (e.g., `v20.x.x` and `10.x.x`). `npm` is the Node Package Manager, which we will use to download Playwright.

### Step 2: Initialize Playwright

Create a brand new folder on your machine where you want to store your framework. Open your terminal and navigate inside that folder.

Run the official Playwright initialization command:

```
npm init playwright@latest
```

This will launch an interactive prompt in your terminal. You should select the following options:

1.  **Do you want to use TypeScript or JavaScript?** 
    *   Select **TypeScript** (Use arrow keys and hit Enter).
2.  **Where to put your end-to-end tests?**
    *   Leave the default as **tests**.
3.  **Add a GitHub Actions workflow?**
    *   Select **false** (We will cover CI/CD much later in the series).
4.  **Install Playwright browsers (can be done manually via 'npx playwright install')?**
    *   Select **true**.

#### What just happened?

When you hit Enter, `npm` automatically downloaded the core Playwright Engine, the `@playwright/test` Runner, and most importantly, **it downloaded the custom browser binaries for Chromium, Firefox, and WebKit.** 

You will never have to manually download a `chromedriver.exe` again!

### Step 3: Understanding the Scaffolding

If you open your project folder in an IDE like **Visual Studio Code**, you will see that Playwright automatically generated a structured framework for you:

#### 1. `package.json`
This is the heart of any Node.js project. It lists your dependencies (Playwright) and allows you to define custom terminal scripts.

#### 2. `playwright.config.ts`
This is the master configuration file for your entire framework. Open it, and you will see configurations for:
*   `testDir: './tests'` (Where your tests live).
*   `retries: 0` (How many times to retry a failed test).
*   `reporter: 'html'` (Playwright has a built-in HTML reporter!).
*   `projects` (An array defining cross-browser execution for Chromium, Firefox, and WebKit).

#### 3. `tests/` Directory
This folder contains `example.spec.ts`. This is a sample test generated by Microsoft to prove that your installation worked. The `.spec.ts` extension stands for "Specification" (a test file) written in TypeScript.

### Step 4: Running the Default Test

Let's execute the sample test to verify everything is working. In your terminal, run:

```
npx playwright test
```

*(Note: `npx` tells Node to execute the Playwright test runner).*

**Output:**

```
Running 6 tests using 4 workers
  6 passed (4.2s)

To open last HTML report run:
  npx playwright show-report
```

Wait, why did it run **6 tests** when there are only 2 scenarios inside `example.spec.ts`?

This is the magic of Playwright! If you look inside your `playwright.config.ts`, you will see that it is configured to run tests across 3 different browsers (Chromium, Firefox, WebKit). Therefore, 2 tests x 3 browsers = 6 total test executions!

### Step 5: Viewing the HTML Report

As the terminal output suggested, Playwright automatically generates a gorgeous HTML report. Run the command to view it:

```
npx playwright show-report
```

This will launch a local web server and open the report in your browser, showing you exactly how long each test took, which browser it ran on, and any failures.

### Conclusion

You have successfully installed Node.js, initialized a Playwright TypeScript project, executed tests across three different browser engines simultaneously, and generated an HTML report—all in less than 5 minutes!
