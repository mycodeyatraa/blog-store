# Handling JavaScript Alerts, Confirm Dialogs, and Prompts in Playwright TypeScript

> 📅 **Last Updated:** 24-Jun-2026

Native browser alerts, confirmation boxes, and prompt boxes pose a unique challenge for automated testing. Because these dialogs are generated by the browser operating system shell rather than the HTML web engine, they do not exist inside the traditional DOM tree. As a result, standard locators like XPath, CSS Selectors, or ID tags will not work.

In this tutorial, we will explore how to intercept, validate, input data, and dismiss browser native alerts using Playwright's event-driven dialog handling model in TypeScript.

---

## 🧭 Native Alert Interaction Flow

In traditional frameworks like Selenium, WebDriver switches context to the alert context. Playwright uses an asynchronous, event-driven model. Before you trigger the action that opens a dialog, you must register a dialog handler callback on the page.

## 🛠️ Step-by-Step Code Walkthrough

Below is the complete Playwright Test implementation. It launches a headless browser, navigates to the overlays portal, and handles:
1. **Simple Alerts**: Reading message content and clicking "OK" (`dialog.dismiss()`).
2. **Confirmation Dialogs**: Clicking "OK" (`dialog.accept()`) and clicking "Cancel" (`dialog.dismiss()`).
3. **Prompt Dialogs**: Injecting a custom text payload using `dialog.accept("Pankaj")` before accepting.

```typescript
import { test, expect } from '@playwright/test';
//
test.describe('Blog 58: Handling Browser Dialogs', () => {
//
  test.beforeEach(async ({ page }) => {
    await page.goto('https://practice.mycodeyatra.com/');
    await page.locator("//span[contains(text(),'Sandbox Arena')]").click();
    await page.locator("//h3[text()='Alerts & Overlays']").click();
  });
//
  test('Accept Alert dialog', async ({ page }) => {
    page.once('dialog', async dialog => {
      expect(dialog.type()).toBe('alert');
      expect(dialog.message()).toBe('This is a standard JavaScript Alert!');
      await dialog.dismiss();
    });
    await page.locator("//button[@data-testid='alert-btn']").click();
    await expect(page.locator("//div[@data-testid='alert-result']")).toContainText('Alert dismissed');
  });
//
  test('Accept Confirm dialog', async ({ page }) => {
    page.once('dialog', async dialog => {
      expect(dialog.type()).toBe('confirm');
      expect(dialog.message()).toBe('Do you want to proceed?');
      await dialog.accept();
    });
    await page.locator("//button[@data-testid='confirm-btn']").click();
    await expect(page.locator("//div[@data-testid='alert-result']")).toContainText('You clicked OK');
  });
//
  test('Dismiss Confirm dialog', async ({ page }) => {
    page.once('dialog', async dialog => {
      expect(dialog.type()).toBe('confirm');
      expect(dialog.message()).toBe('Do you want to proceed?');
      await dialog.dismiss();
    });
    await page.locator("//button[@data-testid='confirm-btn']").click();
    await expect(page.locator("//div[@data-testid='alert-result']")).toContainText('You clicked Cancel');
  });
//
  test('Fill Prompt dialog', async ({ page }) => {
    page.once('dialog', async dialog => {
      expect(dialog.type()).toBe('prompt');
      expect(dialog.message()).toBe('Please enter your name:');
      expect(dialog.defaultValue()).toBe('');
      await dialog.accept('Pankaj');
    });
    await page.locator("//button[@data-testid='prompt-btn']").click();
    await expect(page.locator("//div[@data-testid='alert-result']")).toContainText('Hello, Pankaj!');
  });
//
});
```

## 📊 Summary of Dialog Class Interactions

Playwright's `Dialog` object provides several methods for inspection and interaction. Understanding their support across different dialog types prevents runtime errors:

| Action Method | Simple Alert (`alert()`) | Confirm Box (`confirm()`) | Prompt Box (`prompt()`) |
| :--- | :--- | :--- | :--- |
| `type()` | Returns `'alert'` | Returns `'confirm'` | Returns `'prompt'` |
| `message()` | Returns dialog message | Returns dialog message | Returns dialog message |
| `defaultValue()` | Returns empty string | Returns empty string | Returns default value |
| `accept(text)` | Accepts dialog (ignores text) | Accepts dialog (ignores text) | Types text and accepts dialog |
| `dismiss()` | Dismisses dialog | Dismisses dialog | Dismisses dialog |

## ⚠️ Common Pitfalls

* **Playwright Auto-Dismiss Behavior**: Unlike other frameworks, if you do not register a `dialog` listener, Playwright will automatically **dismiss** any dialog that is triggered. This prevents test execution from hanging indefinitely.
* **Incorrect Event Subscription**: A common mistake is using `page.on('dialog', ...)` which registers a persistent listener that stays active for all subsequent tests. Always use `page.once('dialog', ...)` to bind a handler only to the next expected dialog event.
* **Setting up Listener Too Late**: The `page.once('dialog', ...)` listener must be initialized **before** clicking the button that triggers the dialog. Placing it after the click action will result in the dialog being auto-dismissed before the listener is registered.

## 🙋 Frequently Asked Questions

> **Q: How can we close custom dialog modals that are built with HTML?**
>
> **A:** Custom HTML modal overlays (such as Bootstrap, Tailwind, or React modals) exist inside the DOM. Do *not* use the `dialog` event listener. Instead, interact with them directly using standard locators and click actions.

> **Q: Can we handle HTTP Basic Authentication login prompts using the Dialog API?**
>
> **A:** No. Browser authentication pop-ups are OS-level dialogs and cannot be handled by page-level dialog listeners. Handle them by passing the credentials directly in the URL schema (e.g., `https://username:password@practice.mycodeyatra.com/`) or by setting the HTTP credentials in the browser context options.
