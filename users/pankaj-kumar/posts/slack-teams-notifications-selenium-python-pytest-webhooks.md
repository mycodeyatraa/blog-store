# Slack and MS Teams Notifications: Broadcasting Pytest Results via Webhooks

In our previous article, we generated a beautiful Allure HTML dashboard. But if that dashboard sits silently on a Jenkins server, nobody will ever look at it.

To achieve a true DevSecOps culture, your automation framework must be **noisy**. When a developer breaks the UI, the QA automation suite must immediately broadcast that failure into the company's communication channels.

In this final article of Phase 9, we will learn how to write a custom Pytest script that parses our execution results and sends them directly to a Slack or Microsoft Teams channel via Incoming Webhooks!

---

## 1. What is an Incoming Webhook?

An Incoming Webhook is a unique URL generated by Slack or Microsoft Teams. If you send a simple HTTP `POST` request containing a JSON payload to this URL, the application will automatically render a formatted message in a specific chat channel.

**To get your Webhook URL:**
1. In Slack: Go to App Directory -> "Incoming WebHooks" -> Add to Channel.
2. In Teams: Go to Channel -> Connectors -> "Incoming Webhook".

You will receive a URL that looks like this:
`https://api.example.com/services/T000/B000/EXAMPLE`

## 2. Parsing Pytest Results

We need our Pytest script to count exactly how many tests passed, failed, and skipped.

We can achieve this by hooking into the `pytest_terminal_summary` function in our `conftest.py` file! This hook executes at the very end of the test suite run, granting us access to the final statistics.

**tests/conftest.py**

```python
import os
import requests
# Read the Webhook URL from the Environment Variables for security!
SLACK_WEBHOOK_URL = os.getenv("SLACK_WEBHOOK_URL")
def pytest_terminal_summary(terminalreporter, exitstatus, config):
    """Executes exactly after the final test finishes running."""
    # 1. Extract the metrics
    passed = len(terminalreporter.stats.get('passed', []))
    failed = len(terminalreporter.stats.get('failed', []))
    skipped = len(terminalreporter.stats.get('skipped', []))
    total = passed + failed + skipped
    # 2. Determine if the overall suite passed or failed
    status = "✅ SUCCESS" if failed == 0 else "❌ FAILED"
    color = "#36a64f" if failed == 0 else "#ff0000"
    # 3. If we are running in CI/CD, we want the link to the Allure Dashboard!
    build_url = os.getenv("BUILD_URL", "http://localhost:8080/allure")
    # 4. Construct the Slack JSON Payload
    slack_data = {
        "attachments": [
            {
                "color": color,
                "title": f"UI Automation Suite Execution: {status}",
                "title_link": build_url,
                "text": f"Total Tests: *{total}*\nPassed: *{passed}*\nFailed: *{failed}*\nSkipped: *{skipped}*",
                "footer": "MyCodeYatra Selenium Framework"
            }
        ]
    }
    # 5. Broadcast the message!
    if SLACK_WEBHOOK_URL:
        print("\n[Notifier] Broadcasting results to Slack...")
        response = requests.post(SLACK_WEBHOOK_URL, json=slack_data)
        if response.status_code != 200:
            print(f"[Notifier Error] Failed to send: {response.text}")
    else:
        print("\n[Notifier] SLACK_WEBHOOK_URL not set. Skipping notification.")
```

## 3. Integrating with Jenkins and GitHub Actions

To make this work in your CI/CD pipeline, you must add the Webhook URL to your pipeline secrets.

**In GitHub Actions (`.github/workflows/selenium-tests.yml`):**

```yaml
    - name: Run Pytest Automation Suite
      env:
        SLACK_WEBHOOK_URL: ${{ secrets.SLACK_WEBHOOK_URL }}
        BUILD_URL: "https://github.com/mycodeyatra/repo/actions/runs/${{ github.run_id }}"
      run: |
        pytest tests/ -v
```

**In Jenkins (`Jenkinsfile`):**

```groovy
    environment {
        // Fetch the secret text credential from Jenkins
        SLACK_WEBHOOK_URL = credentials('slack-qa-channel-webhook')
    }
```

## 4. Execution Output

Let's execute the Pytest suite locally and inject our webhook URL to see it in action!

```bash
export SLACK_WEBHOOK_URL="https://api.example.com/webhook/slack/test"
pytest tests/ -s
```

```text

collected 3 items

tests/test_checkout.py::test_visa_checkout FAILED
=================================== FAILURES ===================================
E       AssertionError: Expected 'Success'
=========================== short test summary info ============================
FAILED tests/test_checkout.py::test_visa_checkout - AssertionError: Expected 'Success'
[Notifier] Broadcasting results to Slack...
```

Instantly, the `#qa-automation` Slack channel will receive a beautiful message featuring a red stripe, indicating `Total Tests: 3 | Failed: 1`. 

Because we included the `title_link` inside the JSON payload, the Product Manager can simply click the title of the Slack message, which instantly opens the Allure HTML Dashboard so they can view the screenshot of the failure!

## Conclusion

A silent framework is a useless framework.
- Use the Pytest `pytest_terminal_summary` hook to capture the final execution metrics (Pass, Fail, Skip).
- Construct a rich JSON payload using formatting native to your chat platform (Slack, MS Teams, Discord).
- Use Python's `requests.post()` library to broadcast the metrics to a Webhook URL immediately as the test suite completes.
- **Always** include the `BUILD_URL` so your team can easily navigate from the chat application directly to the Allure Dashboard!
